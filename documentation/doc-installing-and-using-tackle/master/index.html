About Tackle

Tackle is a cloud-native application that assesses your applications to gauge whether the applications are suitable for rehosting on OpenShift Container Platform.

Tackle uses an interactive questionnaire for the assessment. The assessment process enables key stakeholders, led by a Solution Architect, to gather information about applications, to discuss risks flagged by Tackle, and to reach a consensus in formulating recommendations for each application.

Tackle provides the following services in a web console:

Controls

The Controls enable you to manage the following:

  • Stakeholders and stakeholder groups

  • Job functions

  • Business services

  • Tags and tag types

Application Inventory

The Application Inventory enables you to manage your application portfolio and to group applications for assessment.

Reports

Reports display the results of the assessments of applications in the Application Inventory. The reports are dynamic and can be configured to display the results for a filtered list of applications.

Managing the Controls

The Controls page of the Tackle web console enables you to manage the following:

Stakeholder

Individual with an interest in an application or a subject matter expert. Stakeholders are contributors to the application assessment process.

Stakeholder group

Group of related stakeholders. A stakeholder can belong to one or more stakeholder groups. Stakeholder groups are used to assign multiple stakeholders to review an assessment.

Job function

Attribute of a stakeholder. You can define a new job function or use predefined job functions, for example, Business Analyst or Program Manager.

Business service

Attribute of an application, for example, credit card service, transportation, or IT support. A business service is owned by a stakeholder.

Tag

Attribute of an application. Tags are used for creating filters and searches. Each tag is associated with a tag type that determines its display rank and color and enables you to group related tags.

Creating a stakeholder

You can create a stakeholder in the Controls page of the Tackle web console.

Prerequisites

  • The stakeholder must have a valid email address.

Procedure

  1. Click Controls and then click Stakeholders.

  2. Click Create new.

  3. Complete the following fields:

    • Email: Email address of the stakeholder.

    • Display name: Name of the stakeholder.

    • Job function: Optional. You can select a predefined job function or type a job function to create it.

    • Groups: Optional. You can select one or more stakeholder groups from a list of stakeholder groups that you have created.

  4. Click Create.

    The new stakeholder is displayed in the Stakeholders tab of the Controls page.

  5. Expand the stakeholder to view the stakeholder groups to which the stakeholder belongs.

Creating a stakeholder group

You can create stakeholder groups in the Controls page of the Tackle web console. Stakeholder groups enable you to assign multiple stakeholders to review an assessment.

Procedure

  1. Click Controls and then click Stakeholder groups.

  2. Click Create new.

  3. Complete the following fields:

    • Name: Name of the stakeholder group.

    • Description: Optional. Description of the stakeholder group.

    • Member(s): Optional. You can select one or more stakeholders to add to the group.

  4. Click Create.

    The new stakeholder group is displayed in the Stakeholder groups tab of the Controls page.

  5. Expand the stakeholder group to view the group members.

Creating a business service

A business service is an attribute of an application. You can create a business service in the Controls page of the Tackle web console.

Prerequisites

  • You must have added a stakeholder who is the business service owner.

Procedure

  1. Click Controls and then click Business services.

  2. Click Create new.

  3. Complete the following fields:

    • Name: Name of the business service.

    • Description: Optional. Description of the business service.

    • Owner: Select a stakeholder.

  4. Click Create.

    The new business service is displayed in the Business services tab of the Controls page.

Creating a tag type

You can create a tag type in the Controls page of the Tackle web console.

A tag type defines the display rank and color of one or more tags. Tags are applied to applications in order to create filters and groups of applications.

You must create a tag type before you can create a tag.

Procedure

  1. Click Controls and then click Tags.

  2. Click Create tag type.

  3. Complete the following fields:

    • Name: Name of the tag type.

    • Rank: Optional. The rank determines the order in which the tags are displayed in the details view of an application.

    • Color: Optional. You can select a display color.

  4. Click Create.

    The new tag type is displayed in the Tags tab of the Controls page.

Creating a tag

You can create a tag in the Controls page of the Tackle web console.

Tags are applied to applications in the Application Inventory page for filtering and searching applications. Tags are grouped by tag types.

Prerequisites

  • You must have created at least one tag type.

    A tag type defines the display rank and color of one or more tags.

Procedure

  1. Click Controls and then click Tags.

  2. Click Create tag.

  3. Complete the following fields:

    • Name: Name of the tag.

    • Tag type: Select a tag type.

  4. Click Create.

    The new tag is displayed under its tag type in Tags tab of the Controls page.

Managing the Application Inventory

You can manage your application portfolio in the Application Inventory page of the Tackle web console.

Creating an application

You can create an application in the Application Inventory page of the Tackle web console.

Procedure

  1. Click Application Inventory and then click Create new.

  2. Complete the following fields:

    • Name: Name of the application.

    • Description: Optional. Description of the application.

    • Business service: Optional. You can select a business service that describes the application.

    • Tags: Optional. You can select one or more tags from a list of tags that you have defined.

    • Comments: Optional. Comments about the application.

  3. Click Create.

    The new application is displayed in the Application Inventory page.

  4. Expand the application to view its details.

Importing applications

You can import multiple applications into the Application Inventory page of the Tackle web console by using a CSV file.

The CSV file must contain the following fields:

  • Record Type 1: Required field. The value must be 1.

  • Application Name: Required field.

  • Description: Optional.

  • Comments: Optional.

  • Business Service: Optional.

  • Tag Type 1: Optional unless you are creating a tag.

  • Tag 1: Optional. If you create a tag, it must have a corresponding Tag Type.

    You can create Tag Type <number> and Tag <number> fields for additional tags.

Procedure

  1. Click Application Inventory.

  2. Click the Options menu in the toolbar and select Import.

  3. Browse to the CSV file and click Open.

  4. Click Import.

    The imported applications are displayed in the Application Inventory.

Tagging applications

You can tag one or more applications in the Application Inventory page of the Tackle web console.

Prerequisites

  • The Application Inventory must contain at least one application.

  • The Controls must contain at least one tag.

Procedure

  1. Click Application Inventory.

  2. Select one or more applications.

  3. Click the Options menu in the toolbar and select Add tag.

  4. Select one or more tags and then click Apply tags.

  5. Expand an application to view its details.

    The display order of the tags is determined by the Rank of the tag type.

Adding a business service to applications

You can add a business service to one or more applications in the Application Inventory.

Prerequisites

  • The Application Inventory must contain at least one application.

  • The Controls must contain a business service.

Procedure

  1. Click Application Inventory.

  2. Select one or more applications.

  3. Click the Options menu in the toolbar and select Add business service.

  4. Select a business service and then click Add.

  5. Expand an application to its details.

    The business service is displayed.

Managing application dependencies

You can add, delete, and view application dependencies in the Application Inventory page of the Tackle web console.

Prerequisites

  • The Application Inventory must contain at least two applications.

Procedure

  1. Click Application Inventory.

  2. Click the Options menu beside an application and select Manage dependencies.

  3. Select applications in the northbound or southbound dependencies fields.

    To remove dependencies, delete the applications in the dependencies fields.

  4. Click Close.
